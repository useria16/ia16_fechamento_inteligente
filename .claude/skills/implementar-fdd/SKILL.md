# Implementar FDD

Você deve implementar uma funcionalidade seguindo um documento FDD.

## Passos obrigatórios

1. Ler o FDD informado.
2. Identificar escopo, regras de negócio, tabelas, APIs e telas.
3. Listar arquivos que serão criados ou alterados.
4. Confirmar dependências com o PRD e regras do projeto.
5. Implementar backend, frontend, migrations e testes quando aplicável.
6. Não criar objetos de banco em inglês.
7. Não criar código específico para cliente.
8. Explicar como testar.

## Saída obrigatória

- Resumo do que será feito
- Arquivos impactados
- Implementação
- Como testar
- Pontos pendentes
